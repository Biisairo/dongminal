# 모델 선택 · 팀 패턴 카탈로그

## 모델 선택 (`--model` 플래그)

한 세션 전체를 Opus 로 돌리면 비용·지연이 크고, 단순 역할에 Haiku 를 쓰면 품질이 떨어진다. 역할별로 고른다.

| 모델 인자 | 별칭 | 권장 역할 |
|-----------|------|-----------|
| `opus` / `claude-opus-4-7` | Opus 4.7 | 창작 주연, 복잡한 합성·합의, 아키텍처 리더 |
| `sonnet` / `claude-sonnet-4-6` | Sonnet 4.6 | 리서치·코드 리뷰·중간 합성 — 기본 추천 |
| `haiku` / `claude-haiku-4-5` | Haiku 4.5 | 단순 분류/추출, 보조 비평, 경량 워커 |

실제 사용 가능 별칭은 `claude --help` 로 확인. 확실한 방법은 **full model ID** 사용.

기본 선택 가이드:
- 역할 차이가 명확하면 각자 맞는 모델
- 전부 비슷한 리서치면 전원 Sonnet
- 팀장(본인) 은 현재 모델 유지 — 스킬이 바꿀 수 없음

## 팀 패턴 카탈로그

- **Fan-out 리서치**: N명에게 서로 다른 조사 주제 분배 → 병렬 답장 → 종합
- **생성 + 비판 (GAN 스타일)**: A 초안 → B 비판 → A 개정. 2명 팀, 라운드 반복
- **파이프라인**: A(분석) → B(설계) → C(구현) → D(테스트). 각 완료 답장이 다음 팀원 기동 트리거
- **전문 역할**: 고정 역할(frontend/backend/db) 부여
- **Devil's advocate**: 1명에게 "반대 관점 비판"
- **허브 앤 스포크**: 비평가 N명이 각자 수석비평가 A 에게 송신 → A 가 통합 후 작가에게 전달 (수평 협업을 허브로 직렬화)
